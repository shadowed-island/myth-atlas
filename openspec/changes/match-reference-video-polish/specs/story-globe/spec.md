## MODIFIED Requirements

### Requirement: Interactive 3D globe

The system SHALL provide an interactive 3D globe that users can rotate, zoom, and use as the primary story exploration surface while presenting a dense, luminous world-story visual style inspired by the reference video.

#### Scenario: User rotates the globe

- **WHEN** the user drags across the globe
- **THEN** the globe rotates in the drag direction while keeping story nodes and decorative story-density layers attached to their geographic positions

#### Scenario: User zooms the globe

- **WHEN** the user scrolls, pinches, or uses an equivalent zoom gesture
- **THEN** the camera zoom level changes without losing the current geographic focus

#### Scenario: Globe first renders

- **WHEN** the globe scene is visible
- **THEN** the scene includes dark atmospheric depth, luminous grid or scan layers, dense decorative story particles, and highlighted interactive story nodes

### Requirement: Story nodes by location

The system SHALL render story nodes on the globe using latitude and longitude from the story dataset and SHALL supplement them with non-primary decorative points that create visual density without changing search results.

#### Scenario: Stories are loaded

- **WHEN** the story dataset is available
- **THEN** each story with valid coordinates appears as an interactive node at the corresponding globe position

#### Scenario: Story has a category

- **WHEN** a story node is rendered
- **THEN** the node visually reflects the story category using a consistent color, icon, or marker style

#### Scenario: Decorative density is rendered

- **WHEN** the globe scene renders
- **THEN** decorative particles and emoji-style markers appear near story regions without being counted as real story results

### Requirement: Story selection

The system SHALL allow users to select a story from either the globe, the story list, the thumbnail wall, or a guided tour and view its details.

#### Scenario: User selects a globe node

- **WHEN** the user clicks or taps a story node on the globe
- **THEN** the system opens the story detail view for that story

#### Scenario: User selects a list item

- **WHEN** the user selects a story from the explorer panel
- **THEN** the system opens the story detail view and moves the globe focus toward that story location

#### Scenario: Guided tour selects a story

- **WHEN** the guided tour advances to a story
- **THEN** the system opens the story detail view and moves the globe focus toward that story location

### Requirement: Explorer panel synchronization

The system SHALL keep the explorer panel synchronized with globe, filter, search, and guided tour state.

#### Scenario: Visible stories change

- **WHEN** search or filters change the current story set
- **THEN** the explorer panel updates to show the same story set represented on the globe

#### Scenario: Globe focus changes to a region

- **WHEN** the globe is focused on a country or region
- **THEN** the explorer panel highlights or prioritizes stories from that country or region

#### Scenario: Thumbnail wall is visible

- **WHEN** the explorer panel displays story thumbnails
- **THEN** selecting a thumbnail updates the selected story and globe focus

## ADDED Requirements

### Requirement: Opt-in audio atmosphere

The system SHALL provide an explicit sound toggle that enables or disables ambient and interaction audio generated by the browser.

#### Scenario: Sound starts after user action

- **WHEN** the user enables sound
- **THEN** the system starts subtle ambience and interaction sounds without requiring external audio files

#### Scenario: Sound is disabled

- **WHEN** the user disables sound
- **THEN** the system stops ambience and suppresses interaction sounds

### Requirement: Guided story tour

The system SHALL provide a guided tour mode that automatically cycles through representative stories.

#### Scenario: User starts tour

- **WHEN** the user starts the guided tour
- **THEN** the system selects a story, focuses the globe on it, and opens its detail view

#### Scenario: Tour advances

- **WHEN** the tour interval elapses
- **THEN** the system advances to another story and updates globe focus and story details

#### Scenario: User stops tour

- **WHEN** the user stops the guided tour
- **THEN** the system stops automatic story advancement while preserving the current selected story
